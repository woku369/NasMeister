Was ist NasMeister?

NasMeister ist das zentrale Cockpit für alle Node.js-Server-Instanzen auf der Synology NAS. Statt SSH-Terminal, mehreren Browserfenstern und manuellem kill -9 gibt es hier eine einzige Oberfläche mit Ampelsystem.

NasMeister nutzt PM2 als Prozessmanager im Hintergrund und ist selbst eine installierbare PWA (Android & Browser).

Was ist PM2?

PM2 ist ein Prozessmanager für Node.js – vergleichbar mit einem Dienst-Manager speziell für Node-Apps.

Vorteile gegenüber dem bisherigen nohup node server.js &:

  • Automatischer Neustart bei Absturz
  • Start aller Apps nach NAS-Reboot (ersetzt DSM Task Scheduler)
  • Zentrales Log pro App (pm2 logs)
  • CPU/RAM-Monitoring eingebaut
  • Einfaches Kill & Restart per Name statt PID-Suche

Wichtige PM2-Befehle (SSH):

pm2 list # alle Apps + Status pm2 restart zeiterfassung # App neu starten pm2 stop GartenMeister # App stoppen pm2 start LagerMeister # App starten pm2 logs TerminMeister # Live-Log (Strg+C zum Beenden) pm2 logs TerminMeister --lines 50 # letzte 50 Zeilen

Ersteinrichtung – PM2 auf der NAS

Einmalig per SSH auf der Synology ausführen:

1. SSH verbinden (via Tailscale):

ssh wolfg@<tailscale-ip>

2. PM2 installieren:

npm install -g pm2

3. Setup-Script ausführen:

bash /volume1/Gurktaler/nasmeister/pm2-setup.sh

4. Autostart einrichten (den Befehl aus der Ausgabe kopieren und als root ausführen):

pm2 startup # → gibt einen sudo-Befehl aus → diesen ausführen! pm2 save

5. DSM Task Scheduler aufräumen:
Systemsteuerung → Aufgabenplaner → alte Einträge (start_synology.sh) löschen. PM2 übernimmt den Autostart.

NasMeister starten (manuell)

Nach dem PM2-Setup startet NasMeister automatisch. Zum manuellen Start:

ssh wolfg@<tailscale-ip> cd /volume1/Gurktaler/nasmeister npm install # nur beim ersten Mal pm2 start server.js --name nasmeister --env PORT=4000

Erreichbar im Browser / Android:

http://<tailscale-ip>:4000
Als PWA auf Android installieren

NasMeister ist installierbar – kein App Store nötig.

  1. Chrome auf Android öffnen
  2. Adresse eingeben: http://<tailscale-ip>:4000
  3. Menü (⋮) → „Zum Startbildschirm hinzufügen"
  4. Name bestätigen → Hinzufügen

NasMeister erscheint jetzt als App-Icon auf dem Homescreen und startet im Vollbild ohne Browserleiste.

Voraussetzung: Tailscale muss auf dem Android-Gerät aktiv sein.

Neue App hinzufügen

In config.json auf der NAS einen Eintrag ergänzen:

{ "name": "MeineNeueApp", "port": 3007, "dir": "/volume1/Gurktaler/MeineNeueApp", "startScript": "server.js", "healthPath": "/api/health", "color": "#1565C0" }

Danach PM2 registrieren:

pm2 start /volume1/Gurktaler/MeineNeueApp/server.js \ --name MeineNeueApp \ --env PORT=3007 pm2 save

NasMeister neu starten:

pm2 restart nasmeister
Health-Endpunkt zu einer App hinzufügen

Damit NasMeister den Status einer App korrekt anzeigen kann, braucht jede App einen /api/health-Endpunkt. Diesen Snippet in das jeweilige server.js einfügen:

// Health-Check für NasMeister app.get('/api/health', (req, res) => { res.json({ status: 'ok', name: 'AppName', uptime: process.uptime() }); });

Bei reinen Node.js-Servern ohne Express (z. B. mit http.createServer):

if (req.url === '/api/health' && req.method === 'GET') { res.writeHead(200, { 'Content-Type': 'application/json' }); return res.end(JSON.stringify({ status: 'ok' })); }

Nach dem Einfügen App neu starten:

pm2 restart <AppName>
Ampel-Zustände verstehen
🟢 OnlinePM2 meldet „online" UND /api/health antwortet mit 200
🟡 DegradedPM2 meldet „online", aber /api/health antwortet nicht korrekt (App läuft, aber nicht gesund)
🔴 GestopptPM2 meldet „stopped" – App wurde manuell gestoppt
🔴 FehlerPM2 meldet „errored" – App ist abgestürzt
⚫ UnbekanntApp ist nicht in PM2 registriert

Bei „Degraded": App neu starten hilft meist. Wenn das Problem bleibt: Logs prüfen (pm2 logs <name>).

Troubleshooting

NasMeister nicht erreichbar:

  • Tailscale auf dem Gerät aktiv? → Tailscale-App öffnen und verbinden
  • NAS läuft? → DSM-Weboberfläche aufrufen
  • NasMeister-Prozess läuft? → SSH: pm2 list

Alle Apps zeigen „Unbekannt":

  • PM2 noch nicht eingerichtet → pm2-setup.sh ausführen
  • PM2 nicht gefunden → Pfad in config.json oder Env PM2_BIN prüfen

App startet nicht (Fehler-Status):

pm2 logs <AppName> --lines 100

Häufige Ursachen: Port bereits belegt, fehlende node_modules (npm install im App-Verzeichnis ausführen), Syntax-Fehler nach einem Update.

Nach NAS-Neustart laufen die Apps nicht:

pm2 startup # → gibt sudo-Befehl aus → ausführen pm2 save # → speichert aktuelle Prozessliste
